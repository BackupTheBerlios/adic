ADIC

Any door is closed.

Description

ADIC (Any door is closed) is a cooperative multi-player game. The goal is to lock all the players of the other teams. Main development platform is GNU/Linux - but now there are cross-compiled binaries for win32. Beside of playing the game you can write a bot and use the game as test bed for AI/robotics algorithms.

News/Changelog:

Version 0.9.0 (2003-03-30)

  • tuned parameters (due to popular demand - gameplay is faster now => more action)
  • added music from Olech Kaczanowski (amuzen)

It creates on the fly some cpp's that include other cpps (=>this reduces the overhead of parsing the headers again and again - it seems gcc does not have precompiled headers yet?). This works as long as .cpp files don't assume they are one compilation unit (=> no static variables, no obscure macros, ...)

  • SDL_mixer >=1.2

Running

  • Take a look at the meta-server to find a running server
  • if the list is empty start your own server. Run adicserver (perhaps you want to disable the metaserver and choose another map. f.e: adicserver --metaServer="" --meshURI=data:mesh.xml, see adicserver --help for more info)
  • run ./adicclient (perhaps you need to set --libGL and perhaps you want to set --quality=1 if you have hardware accelerated OpenGL, run adicclient --help for more info)
  • to start a bot team run adicbot --implementation=i (Where i is currently "KarmeBot" or "BlackBot")

Gameplay

Goal

ADIC is a cooperative multiplayer game. The goal is to lock all players of the other teams. A player is locked if all doors of the room he is in are closed. A locked player can't move.

The characters

The characters differ in speed, mass, endurance, ...

Fitness

Either embrace your team mates, drink some water from the springs or just take a rest to recover.

  • Q: It does look ugly !
  • A: Try the --quality=1 switch or enable line smoothing with the "l" key. And change the resolution
